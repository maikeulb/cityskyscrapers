Restful API for NYC landmarks, powered by Flask + PostgreSQL

Features
Searching, Sorting, Ordering
Paging
Status Codes
Exc - in progress
HTTP Response Caching

Quickstart
First, create an nyclandmarks postgreSQL database and configure config.py so that it points
to your instance, then set FLASK_APP to nyclandmarks.py.
$ pip install -r requirements.txt
$ flask db upgrade
$ flask seed-db
$ flask run
$ localhost:5000/api/boroughts

----
Alternatively:
$ docker-compose up
$ docker-compose build
$ localhost:8000/api/boroughts

Routes
GET       api/boroughs          Return the collection of all boroughs
GET       api/boroughs/<int:id> Return a borough
POST      api/boroughs          Create a borough
PUT       api/boroughs/<int:id> Update a borough
PUT       api/boroughs/<int:id> Partial update a borough
DELETE    api/boroughs/<int:id> Delete a borough

GET       api/boroughs/<int:boroughId>/landmarks          Return the collection of all the  borough landmarks
GET       api/boroughs/<int:boroughId>/landmarks/<int:id> Return a borough landmark
POST      api/boroughs/<int:boroughId>/landmarks          Create a borough landmark
PUT       api/boroughs/<int:boroughId>/landmarks/<int:id> Update a borough landmark.
PATCH     api/boroughs/<int:boroughId>/landmarks/<int:id> Partially update a borough landmark
DELETE    api/boroughs/<int:boroughId>/landmarks/<int:id> Delete a borough landmark
